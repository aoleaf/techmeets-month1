## 学んだこと

### PHP

- **`$_POST` でフォームの値を受け取る**  
  `$_POST["name"]` のようにキー名で取得できる。フォームが送信されたかどうかは `$_SERVER["REQUEST_METHOD"] === "POST"` で判定する。

- **hidden フィールドで画面間のデータを引き継ぐ**  
  PHP はリクエストをまたいで変数を保持しない（ステートレス）。画面遷移で値を保持するには `<input type="hidden">` を使って次の POST に含める。

- **`??`（null 合体演算子）でデフォルト値を設定する**  
  `$name = trim($_POST['name'] ?? '');` のように書くと、`$_POST['name']` が存在しない場合に空文字を使える。`isset()` と三項演算子を組み合わせた書き方をワンライナーにできる。

- **`trim()` でバリデーション前に入力値を正規化する**  
  スペースや改行だけを入力した場合に空チェックをすり抜けないよう、取得直後に `trim()` をかけておく。

- **`filter_var()` でメール形式を検証する**  
  `filter_var($email, FILTER_VALIDATE_EMAIL)` で RFC 準拠の形式チェックができる。自前で正規表現を書くより安全で読みやすい。

- **XSS 対策は「出力時」に `htmlspecialchars()` をかける**  
  入力時に加工するとメール送信やDB保存で二重エスケープが起きる。HTML として出力する直前にエスケープするのが正しいタイミング。

### JavaScript

- **`const` と `let` の使い分け**  
  変更しない値は `const`、後で更新する値は `let` を使う。意図しない再代入を防げる。

- **`forEach` で配列を処理する**  
  `items.forEach(function(item) { ... })` で配列の要素を1件ずつ取り出せる。`for` 文より宣言的に書けて読みやすい。

- **DOM操作でページを動的に更新する**  
  `document.getElementById()` で要素を取得し、`innerHTML` や `textContent` で書き換える。`createElement()` + `appendChild()` で要素を動的に追加できる。

- **`submit` イベントと `click` イベントの違い**  
  ボタンの disabled 化は `onclick` ではなく `form.addEventListener('submit', ...)` で行う。`onclick` で先に disabled にするとブラウザがフォーム送信をキャンセルするケースがある。

## 難しかった点

- **ステップ遷移の設計**  
  3画面を1ファイルで実装するとき、`$step` の値によって表示内容と処理を切り替える構造が最初はわかりにくかった。「POST されてきた `step` の値を見てから、次の `step` を決める」という流れを図で整理してようやく理解できた。

- **エスケープのタイミング**  
  「入力時にエスケープすればいい」と思っていたが、出力時にかけるのが正しいと知った。入力値を別の用途（メール送信など）にも使う場合に問題が起きる理由も理解できた。

## 次に活かしたいこと

- **バリデーションは必須項目をすべて洗い出してからコードに落とす**  
  `comment.php` では `$comment` の空チェックを書き忘れた。実装前にフォームの入力項目をリストアップして、各項目に必要なチェックを一覧にしておく。

- **ユーザー入力を出力するときは必ず `htmlspecialchars()` をセットで書く癖をつける**  
  「あとでエスケープを足す」は忘れやすい。`$_POST` から取得した値を出力するコードを書く瞬間にエスケープもセットで書く。

- **`===` での厳密比較を優先する**  
  `== ""` よりも `=== ''` のほうが型の意図が明確になる。PHP の型強制による予期しない挙動を防ぐため、比較には `===` を使う習慣をつける。

- **hidden フィールドはサーバー側で再バリデーションする**  
  hidden フィールドの値はブラウザの開発者ツールで書き換えられる。確認画面 → 送信のフローでも、サーバー側でバリデーションを再実行することが本番では必要になる。
